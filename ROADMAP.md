# Codex Pet Club 迭代路线

首个版本坚持一条主链路：用户浏览桌宠、复制唯一 ID，再由官方 Skill 安装。公开投稿、账号系统和在线审核暂不进入上线范围。

## 当前基线

- 两只已审核桌宠已进入版本化目录
- 列表展示待机动作，详情展示全部标准动作
- ZIP 存在 EdgeOne Blob，网页没有包直链
- `/admin`、管理 API 和投稿 API 在公网关闭
- Next.js 构建、发布契约测试和 EdgeOne 公网烟测已通过

## 迭代顺序

### P0：完成首版上线

1. **绑定稳定自定义域名**
   - 使用 EdgeOne 自定义域名替代三小时有效的系统预览链接
   - 将 Skill 默认 registry 指向正式 HTTPS 域名
   - 验证国内网络访问、目录、预览、包下载和 404/403 门禁

2. **固化发布流程**
   - 用脚本完成桌宠校验、打包、Blob 上传、目录更新和回滚
   - GitHub Actions 继续执行 lint、build、test
   - 部署后自动做 API、哈希和 Skill 安装烟测

### P1：提高安全性和可维护性

3. **升级 Skill 下载授权**
   - 当前请求头只表达“通过 Skill 操作”，不能阻止伪造客户端
   - 如果需要真正的访问控制，改用短期签名下载地址或服务端令牌

4. **桌宠版本和撤回机制**
   - 为桌宠增加版本号、兼容范围、下架状态和旧版回滚
   - 保留发布记录与 Blob 对象校验结果

5. **重新开放投稿和审核**
   - 先增加管理员认证、审核审计、上传限流和 ZIP 防护
   - 再开放 Skill 投稿，避免把账号系统提前塞进首版

### P2：改善使用体验

6. 独立桌宠详情 URL、分享元信息和动作控制
7. 分类、标签、搜索、排序和分页
8. 创作者主页、投稿状态和版本更新提示

### P3：社区治理

9. 举报、版权申诉、下架和恢复流程
10. 安装量、精选和自动审核辅助

## 下一步

当前只做第 1 项：拿到并绑定一个正式域名。域名生效后再做第 2 项，不提前建设账号、在线审核或复杂数据库。
