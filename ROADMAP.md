# Codex Pet Club 迭代路线

产品主链路保持简单：匿名用户浏览桌宠并通过官方 Skill 安装；创作者在第一次投稿或管理作品时注册账户，将 Skill Key 绑定到本地，之后的操作由 Skill 自动携带身份。

## 已完成基线

- v0.1：Cloudflare Workers、私有 R2、公开桌宠库和 Skill-only 下载闭环
- v0.2：版本化发布、下架与恢复、目录校验和部署烟测
- v0.3：在线投稿审核、管理员认证、审计日志、限流、备份和恢复预检
- v0.3.3：真实动作预览、列表与详情分离、稳定公网地址和用户上传验证
- v0.4.0：邮箱验证码账户、最多 3 个 Skill Key、永久用户 ID 和账户数据备份
- v0.4.1：Skill Key 绑定、身份查询、本机解绑和强制归属账户的投稿
- v0.4.2：每次调用自动检查版本、官方 Release 校验、事务升级与失败回滚
- v0.4.3：跨设备“我的投稿”、状态筛选、详情查询和账户页投稿管理
- v0.4.4：审核结果邮件、失败重试、管理端投递状态和人工重发
- v0.4.5：真实备份验证、过期数据清理、定时生产烟测、隐私与投稿规则、统一发布流程

## 已完成：v0.4.0 账户与作品归属

目标：先建立最小可用身份基础，不把社交功能、复杂权限或密码系统塞进首版。

1. **邮箱验证码账户**
   - 注册与登录使用同一套邮箱验证码流程
   - 新用户首次验证时填写创作者名称并创建永久用户 ID
   - 邮箱不公开，只用于登录、审核结果和必要服务通知
   - 浏览器会话为 HttpOnly、Secure、SameSite Cookie
   - 验证码 10 分钟失效、最多尝试 5 次，并按邮箱和来源地址限流

2. **Skill Key 管理**
   - 用户在 `/account` 创建、查看状态和撤销 Key
   - 每个账户最多 3 个有效 Key，撤销后释放额度
   - Key 原文只展示一次，服务器只保存哈希
   - 同一个 Key 可在多台电脑使用；用户也可按设备拆分 Key
   - 撤销后使用该 Key 的所有设备立即失效

3. **作品绑定用户 ID**
   - 投稿携带有效 Key 时自动写入 `owner_user_id`
   - 展示作者名称取账户资料，不信任客户端自报作者
   - 更换或撤销 Key 不改变历史作品归属
   - v0.4.0 曾短期兼容旧版匿名投稿，v0.4.1 起新投稿必须携带有效 Key

4. **数据安全与运维**
   - D1 增加用户、验证码、会话、Key 和认证限流表
   - R2 备份 schema v2 增加用户与 Key 哈希，并兼容 schema v1 恢复
   - 增加迁移执行和账户备份恢复自动化测试
   - 生产依赖 `AUTH_SECRET`、自托管邮件服务 URL 和独立服务 Token
   - 邮件服务复用服务器现有 SMTP 配置，只发送固定验证码模板并执行第二层限流

## 已完成：v0.4.1 Skill 身份绑定

目标：让用户只复制一次 Key，之后无需在每次命令里重复提供身份。

1. 增加“绑定桌宠库 Key”“查看当前身份”“解除绑定”命令
2. Key 写入用户本地 Codex 配置目录，不进入桌宠源码、日志或聊天回复
3. 每次上传和作品管理请求自动携带 `Authorization: Bearer <Key>`
4. 绑定时调用 `/api/me` 校验 Key，并显示创作者名称和脱敏邮箱
5. 对无效、撤销、超额和未绑定状态给出可执行的中文错误提示
6. 已完成 Windows 旧配置兼容、stdin 安全配置和防泄漏自动化测试

v0.4.1 已停止接受新的匿名投稿；公开浏览和桌宠安装仍不要求登录。

## 已完成：v0.4.2 Skill 自动更新

目标：用户只安装一次，以后每次使用 Skill 都自动对齐官方最新版。

1. 每条 CLI 命令执行前查询桌宠库版本清单，离线时继续使用当前版本
2. 只接受版本号完全匹配的官方 GitHub Release 地址
3. 下载后校验文件大小、SHA-256、目录穿越、重复路径、符号链接和运行文件完整性
4. 只替换标准安装目录，不自动修改开发者的源码工作区
5. 在同一磁盘事务切换；失败恢复当前版本，成功立即删除临时旧目录，不保留旧 Skill 副本
6. 配置、账户 Key、桌宠安装记录和备份保存在 Skill 目录外，升级不受影响
7. 升级完成后停止当前命令，提示用户下一轮重试，以加载新的 `SKILL.md`
8. GitHub Release 使用可重复构建 ZIP，线上烟测同时验证清单与实际 Release SHA-256

## 已完成：v0.4.3 我的投稿

目标：用户能追踪自己提交的内容，不再依赖手工保存投稿 ID。

1. Skill 本地记录投稿 ID，并提供“查看我的投稿”和“查看最近一次投稿”
2. 服务端按用户 ID 查询投稿，不能仅凭投稿 ID 查看他人未公开内容
3. 返回待审核、已通过、已拒绝、已下架状态和审核备注
4. 网站账户页增加投稿列表、状态筛选和桌宠详情入口
5. 投稿详情接口要求浏览器会话或 Skill Key，并验证作品归属
6. 本地记录只作为便捷缓存，跨设备查询始终以服务器数据为准

## 已完成：v0.4.4 审核结果通知

目标：用户不需要反复查询，也能及时知道投稿审核结果。

1. 审核通过、拒绝或下架后，向账户邮箱发送简短通知
2. 邮件仅包含作品名称、状态、审核说明和返回账户页的链接，不附带 Key
3. 保存通知投递状态并支持有限重试，避免一次邮件故障阻塞审核操作
4. 管理后台显示通知是否成功，失败时允许人工重发
5. 为改邮箱、Key 全部丢失和账户停用预留人工恢复流程

## 已完成：v0.4.5 上线收口

目标：在继续扩展功能前，先让首版能够稳定、透明地交给真实用户。

1. 每日备份后立即完成恢复预检，只有验证通过才清理过期数据
2. 自动清理过期验证码、会话、认证限流、投稿限流和邮件限流记录
3. 管理后台展示最近维护状态，并支持人工执行相同维护流程
4. 发布隐私说明、投稿规则以及举报和版权反馈入口
5. GitHub Actions 每日执行公开生产烟测，真实邮件烟测保留人工触发
6. 统一生产发布顺序为检查、备份、迁移、部署和烟测，并保留明确回滚入口

网站系统代码已采用 MIT License；用户投稿作品、角色形象和图集不自动纳入该许可。

## 已完成：v0.5.0 审核效率

目标：投稿量增加后，管理员仍能快速且一致地完成审核。

1. 审核队列按状态筛选，并搜索名称、作者、桌宠标识、投稿 ID 和 SHA-256
2. 根据历史投稿的桌宠标识和包哈希显示精确重复提示
3. 支持只查看存在历史重复的投稿，并提供审核队列分页
4. 固化动作图集、内容质量、授权来源和元数据四项审核清单
5. 提供常用拒绝与下架原因模板，同时保留人工补充说明
6. 审核决定继续写入审计日志，保留时间、状态和原因

## v0.5.1 桌宠库扩展

目标：桌宠数量增加后仍容易发现、比较和安装。

1. 分类、标签、关键词搜索、最新/热门排序和分页
2. 独立可分享详情 URL、社交分享元信息和作者主页
3. 详情页增强动作切换、播放速度、版本历史和更新说明
4. Skill 支持按分类浏览、搜索桌宠和检查已安装桌宠更新
5. 记录隐私友好的安装量与失败率，不记录用户桌面内容

## v0.6.0 稳定运营

目标：从“可用”进入“可以持续运行和恢复”。

1. API、邮件、D1、R2 和定时备份异常告警
2. 定期执行备份恢复演练，并记录最近一次成功时间
3. 统计注册、Key 激活、投稿、审核时长和安装成功率
4. 增加 R2 容量、D1 行数、邮件额度和错误率预警
5. 建立举报、版权申诉、用户停用、Key 全部撤销和数据导出流程
6. 持续复核数据保留周期、隐私说明、服务条款和许可证边界

## 暂不进入首轮的内容

- 用户名加密码登录、第三方 OAuth、双因素认证
- 关注、评论、点赞、私信等社交功能
- 复杂团队账户和多级权限
- 付费、订阅和桌宠交易
- 未经人工确认的自动发布

这些能力只有在真实用户量和运营需求证明有必要后再排期，避免过早增加维护成本。
