# Codex Pet Club 迭代路线

首个版本坚持一条主链路：用户浏览桌宠、复制唯一 ID，再由官方 Skill 安装。公开投稿、账号系统和在线审核暂不进入上线范围。

## 当前基线

- 两只已审核桌宠已进入版本化目录
- 列表展示待机动作，详情展示全部标准动作
- ZIP 存在私有 Cloudflare R2，网页没有包直链
- `/admin`、管理 API 和投稿 API 在公网关闭
- Cloudflare Workers 使用稳定的 `*.workers.dev` 平台地址，无需自定义域名

## 迭代顺序

### 已完成：v0.1 公网上线

1. Cloudflare Workers、私有 R2 和正式 Skill 下载闭环
2. 两只真实桌宠、动作预览和唯一 ID 安装
3. 公网管理路由与投稿接口关闭

### 当前：v0.2 发布运营

1. **固化发布流程**
   - 一条命令完成校验、打包、R2 上传、部署和烟测
   - 每次生产变更必须显式使用 `--deploy`
   - GitHub Actions 继续执行目录校验、lint、build 和 test

2. **版本和撤回机制**
   - 稳定 ID 下保存不可变版本与当前版本
   - 下架不删除历史包，恢复可选择任意已发布版本
   - 保留发布、下架和恢复原因

### P1：提高安全性和可维护性

3. **升级 Skill 下载授权**
   - 当前请求头只表达“通过 Skill 操作”，不能阻止伪造客户端
   - 如果需要真正的访问控制，改用短期签名下载地址或服务端令牌

4. **重新开放投稿和审核**
   - 先增加管理员认证、审核审计、上传限流和 ZIP 防护
   - 再开放 Skill 投稿，避免把账号系统提前塞进首版

### P2：改善使用体验

5. 独立桌宠详情 URL、分享元信息和动作控制
6. 分类、标签、搜索、排序和分页
7. 创作者主页、投稿状态和版本更新提示

### P3：社区治理

8. 举报、版权申诉和治理流程
9. 安装量、精选和自动审核辅助

## 下一步

完成 v0.2 发布运营闭环后，下一项是管理员认证与重新开放投稿。账号、社交功能和复杂数据库继续延后。
